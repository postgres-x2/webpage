Postgres-XC Roadmap

Current Limitations of Postgres-XC

At present, Postgres-XC provides major transaction management features similar to PostgreSQL, except for savepoints.

On the other hand, Postgres-XC needs to enhance support for general statements.
As of Version 0.9.3, Postgres-XC supports statements which can be executed on a single data node, or on multiple nodes for single and multi step.
This new version adds support for:

  • Cursor Support
  • Basic cross-node operation
  • Global timestamp
  • DDL synchronisation
  • Cluster-wide installer
  • Cluster-wide operation utilities
  • Driver support (ECPG, JDBC, PHP, etc.)
  • Extended Query Protocol (for JDBC)
  • Support of external 2PC from application
However there are some limitations please refer to SQL Limitations document for further details.

There is no support yet for SELECT in FROM clause.

We will be expanding the coverage of supported SQL and as well High-Availability (HA) features in the coming months.

Upcoming Releases and Features

Current plan of future releases and features are as follows:

Version 0.9.4 (March, 2011)

  • License modified from LGPL to BSD
  • Merge with PostgreSQL 9.0.3
  • HA Capability
    • Datanode mirroring functionality: multiple Datanodes seen as only one node,
      SQL-based replication
    • XC Cluster Manager: manage in an allocated shared memory space node information,
      used to report node failures
    • XC watcher: Central part managing the XC Cluster.
      It contains commands to start/stop/restart the cluster.
      It is also in charge of gathering and reporting to XCM entities nodes that failed.
  • Bug corrections

After Version 0.9.4, on the way to Version 1.0

  • PITR cluster-wide recovery
  • Trigger
  • Temporary (TEMP) tables
  • HOT standby full support
  • Cross-node operation optimization
  • Multi-step Prepared statments
  • More variety of statements
  • GTM-Standby
  • Savepoint
  • Session Parameters
  • Backward cursor
  • Batch, statement pushdown
  • Global constraints
  • Tuple relocation (distribute key update)
  • Regression tests
