Postgres-XC Roadmap

Current Limitations of Postgres-XC

At present, Postgres-XC provides major transaction management features similar to PostgreSQL, except for savepoints.

On the other hand, Postgres-XC needs to enhance support for general statements.
As of Version 0.9.5, Postgres-XC supports statements which can be executed on a single data node, or on multiple nodes for single and multi step.
This new version adds support for:

  • License changes to PostgreSQL license
  • PITR and barrier mechanism (create consistent recovery points in a cluster of nodes)
  • GTM standby
  • GROUP BY
  • Single-step PL/PGSQL function support
  • ROLE DDL, REVOKE, GRANT, etc.
  • Connection pooling supports multiple users
  • Session parameters
  • Improvement of aggregates
  • New types of distribution keys for table distribution
  • Cluster setting simplified
  • GTM proxy reconnection
  • Merge with PostgreSQL 9.0.4
  • pg_regress
  • DBT-1 stabilization
  • More code stabilization
However there are some limitations please refer to SQL Limitations document for further details.

There is no support yet for SELECT in FROM clause.

In the next months, we will expand HA features with GTM standby improvement and synchronous streaming replication. Another focus is the support of multi-prepared statements or complicated statements such as subqueries, SELECT INTO, etc.

Upcoming Releases and Features

Current plan of future releases and features are as follows:

Version 0.9.6 (September, 2011)

  • GTM standby improvements
  • HAVING
  • Extension of synchronous streaming replication
  • Creation of tables in subset of nodes
  • Entension of query planning with remote node scan extensions
  • Bug corrections
  • Performance improvement

After Version 0.9.6, on the way to Version 1.0

  • Standby node method based on PostgreSQL's HOT Standby
  • Trigger
  • SELECT INTO
  • SSL connection protocol for pooler (secure your cluster!)
  • Temporary (TEMP) tables
  • HOT standby full support
  • Cross-node operation optimization
  • Multi-step Prepared statments
  • More variety of statements
  • GTM-Standby
  • Savepoint
  • Backward cursor
  • Batch, statement pushdown
  • Global constraints
  • Tuple relocation (distribute key update)
  • Regression tests
