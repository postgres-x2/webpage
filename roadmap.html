Postgres-XC Roadmap

Current Limitations of Postgres-XC

At present, Postgres-XC provides major transaction management features similar to PostgreSQL, except for savepoints.

As of Version 1.0beta2, Postgres-XC supports statements which can be executed and support for identification of expressions non-shippable to remote nodes.
This new version is centralized on stabilization and adds support for:

  • Fast query shipping (FQS)
  • SERIAL
  • TABLESPACE
  • Rules
  • Utility to clean up 2PC transactions in cluster (pgxc_clean)
  • Utility for initialization of GTM (initgtm)
  • Relation-size functions
  • All 139 regressions passing
  • Support of "make world"

Currently in beta1 state, the next effort to a stable version will be stabilization and bug fix.

Upcoming Releases and Features

Current plan of future releases and features are as follows:

Next release 1.0.0

  • Fast query shipping (FQS)
  • SERIAL
  • TABLESPACE
  • Utility to clean up 2PC transactions in cluster (pgxc_clean)
  • Utility for initialization of GTM (initgtm)
  • Relation-size functions

  • Stable regressions

Next release V1.1 (October 2012)

  • Data redistribution
  • Tuple relocation (distribute key update)
  • Online server addition and removal
  • Cluster bootstrap

Beyond 1.1

  • SSL connection protocol for pooler (secure your cluster!)
  • HOT standby full support
  • Cross-node operation optimization
  • Savepoint
  • Batch, statement pushdown
  • Global constraints
